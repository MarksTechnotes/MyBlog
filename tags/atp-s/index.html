Generate and Store Vector Embeddings in Autonomous Database 26ai

Overview

Oracle Autonomous Database 26ai introduces native Vector storage and Vector search capabilities, making it possible to combine similarity search with traditional relational keyword search directly inside the database.

In Part 1 of this series, the focus is on preparing the data and storing embeddings inside ATP-S 26ai.

This includes:

  • Loading a Kaggle dataset
  • Creating relational and vector tables
  • Concatenating review text into a single embedding input
  • Generating embeddings externally using Hugging Face
  • Loading embeddings back into Oracle for vector search

In Part 2 we will perform a combined semantic and relational search in 26ai on this vector data and finally in Part 3 we will extend the search functionality to hybrid deployments where the relational data exists in 19c instead of 26ai.
